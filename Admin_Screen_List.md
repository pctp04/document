# Screen List

## Admin Branch Screens

| Screen | Description | REMARKS |
|--------|-------------|---------|
| **1.0 Login** | User interface for validation of admin credentials. Admins can login with username and password. | Basic authentication system. Session-based authentication. Redirects admin to Admin Dashboard after successful login. Supports "remember me" functionality for 30-day session persistence. No CAPTCHA currently implemented. |
| **2.0 Admin Dashboard** | Main dashboard for administrators displaying system overview statistics and quick actions. | Displays total teachers, total students, total subjects, and total curriculums. Shows active/inactive counts for each entity. Includes grade distribution chart (Grade 7-10), system status indicators, and quick action buttons to navigate to management screens. Provides navigation to all management sections. |
| **3.0 Teacher Management** | User interface for viewing, creating, editing, and deleting teachers in the system. | Lists all teachers organized by filters (name, status, age, ID). Displays teacher details including name, username, ID, age, address, and assigned subjects. Provides options to create, edit, view details, or delete teachers. Includes filtering, sorting, pagination, and export functionality (PDF, Excel, CSV, Image). Supports subject assignment to teachers. All operations include validation and confirmation dialogs where appropriate. |
| **4.0 Student Management** | User interface for viewing, creating, editing, and deleting students in the system. | Lists all students with their details including name, ID, age, grade level, and address. Displays profile pictures when available. Shows grade level distribution. Provides options to create, edit, view details, or delete students. Includes filtering by grade level and status, sorting, pagination, and export functionality (PDF, Excel, CSV, Image). All operations include validation and confirmation dialogs where appropriate. |
| **5.0 Subject Management** | User interface for viewing, creating, editing, and deleting subjects in the system. | Lists all subjects with their details including subject code, subject name, and description. Displays active/inactive status. Provides options to create, edit, view details, or delete subjects. Includes filtering by code and status, sorting, and export functionality (PDF, Excel, CSV, Image). Validates subject code uniqueness. All operations include validation and confirmation dialogs where appropriate. |
| **6.0 Curriculum Management** | User interface for viewing, creating, editing, and deleting curriculum programs in the system. | Displays all curriculums with their details including curriculum name, code, academic year, semester, grade level, and description. Shows statistics for each curriculum including total subjects, teachers, and enrolled students. Provides options to create, edit, view details, or delete curriculums. Includes filtering by grade level, academic year, and status. Supports assigning subjects with teachers and enrolling students to curriculums. All operations include validation and confirmation dialogs where appropriate. |

## Teacher Branch Screens

| Screen | Description | REMARKS |
|--------|-------------|---------|
| **7.0 Teacher Login** | User interface for validation of teacher credentials. Teachers can login with username and password. | Basic authentication system. Session-based authentication. Redirects teacher to Teacher Dashboard after successful login. Supports "remember me" functionality for 30-day session persistence. |
| **8.0 Teacher Dashboard** | Main dashboard for teachers displaying their assigned classes, schedules, and notifications. | Displays overview of teacher's assigned curriculums, upcoming classes, pending assignments, and recent messages. Shows statistics for each assigned curriculum including total students, average grades, and total grades. Provides navigation to course management, assignment management, student performance, and communication center. |
| **9.0 Manage Courses (Teacher)** | User interface for teachers to view and manage their assigned courses/curriculums. | Lists all curriculums assigned to the logged-in teacher. Displays curriculum details including name, code, academic year, semester, grade level, and enrolled students. Allows teachers to view course materials and student lists. Provides navigation to assignment management and student performance for each course. |
| **10.0 Manage Assignments (Teacher)** | User interface for teachers to create, distribute, and grade assignments for their courses. | Displays assignments organized by curriculum. Allows teachers to create new assignments with title, description, due date, and attached files. Supports creating assignments for all students in a curriculum at once. Enables teachers to view submitted assignments, grade them, and provide feedback. Includes assignment status tracking (draft, published, closed). |
| **11.0 Student Performance (Teacher)** | User interface for teachers to view and analyze student grades and attendance. | Displays student performance metrics organized by curriculum. Shows student name, attendance record, grades for assignments, and overall performance. Allows teachers to input grades and remarks. Provides filtering by curriculum, student name, and date range. Includes grade calculation and totals. Supports export functionality for performance reports. |
| **12.0 Communication Center (Teacher)** | User interface for teachers to send messages to students and parents. | Allows teachers to compose and send messages to individual students, groups of students, or parents. Displays message inbox with received messages. Supports message attachments. Includes message status tracking (sent, read, unread). Provides filtering and search functionality for messages. |

## Student Branch Screens

| Screen | Description | REMARKS |
|--------|-------------|---------|
| **13.0 Student Login** | User interface for validation of student credentials. Students can login with username and password. | Basic authentication system. Session-based authentication. Redirects student to Student Dashboard after successful login. Supports "remember me" functionality for 30-day session persistence. |
| **14.0 Student Dashboard** | Main dashboard for students displaying enrolled courses, upcoming assignments, and notifications. | Displays overview of student's enrolled curriculums, upcoming assignments with due dates, recent messages, and attendance summary. Shows statistics including total enrolled courses, pending assignments, and overall grade average. Provides navigation to course materials, assignment submission, attendance record, and messaging center. |
| **15.0 Course Materials (Student)** | User interface for students to access course-related documents and resources. | Lists all enrolled curriculums with available course materials. Displays material title, description, upload date, and download link. Organizes materials by curriculum and subject. Allows students to download course documents, lecture notes, and other resources. Includes search and filter functionality. |
| **16.0 Assignment Submission (Student)** | User interface for students to submit assignments and view grades. | Displays assignments organized by curriculum with due dates and status. Allows students to submit assignments by uploading files before the due date. Shows submission history with submission date, uploaded files, grade received, and teacher feedback. Displays assignment status (not started, in progress, submitted, graded). Includes validation to prevent late submissions. |
| **17.0 Attendance Record (Student)** | User interface for students to view their personal attendance history. | Displays attendance records organized by curriculum and date. Shows date, course/subject name, and attendance status (present, absent, late, excused). Provides attendance statistics including total days, present days, absent days, and attendance percentage. Includes filtering by curriculum, date range, and status. Supports export functionality for attendance reports. |
| **18.0 Messaging Center (Student)** | User interface for students to communicate with teachers and peers. | Allows students to send messages to teachers, receive messages from teachers, and communicate with peers. Displays message inbox with received messages organized by sender and date. Supports message attachments. Includes message status tracking (sent, read, unread). Provides filtering and search functionality for messages. |
