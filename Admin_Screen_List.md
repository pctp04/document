# Screen List

## Admin Branch Screens

| Screen | Description | REMARKS |
|--------|-------------|---------|
| **1.0 Login** | Unified user interface for validation of user credentials. Users (Admin, Teacher, or Student) can login with username and password. | Unified role-based authentication system. Session-based authentication. After successful login, users are redirected to their respective dashboards based on role (Admin → Admin Dashboard, Teacher → Teacher Dashboard, Student → Student Dashboard). Supports "remember me" functionality for 30-day session persistence. No CAPTCHA currently implemented. |
| **2.0 Admin Dashboard** | Main dashboard for administrators displaying system overview statistics and quick actions. | Displays total teachers, total students, total subjects, and total curriculums. Shows active/inactive counts for each entity. Includes grade distribution chart (Grade 7-10), system status indicators, and quick action buttons to navigate to management screens. Provides navigation to all management sections. |
| **3.0 Teacher Management** | User interface for viewing, creating, editing, and deleting teachers in the system. | Lists all teachers organized by filters (name, status, age, ID). Displays teacher details including name, username, ID, age, address, and assigned subjects. Provides options to create, edit, view details, or delete teachers. Includes filtering, sorting, pagination, and export functionality (PDF, Excel, CSV, Image). Supports subject assignment to teachers. All operations include validation and confirmation dialogs where appropriate. |
| **4.0 Student Management** | User interface for viewing, creating, editing, and deleting students in the system. | Lists all students with their details including name, ID, age, grade level, and address. Displays profile pictures when available. Shows grade level distribution. Provides options to create, edit, view details, or delete students. Includes filtering by grade level and status, sorting, pagination, and export functionality (PDF, Excel, CSV, Image). All operations include validation and confirmation dialogs where appropriate. |
| **5.0 Subject Management** | User interface for viewing, creating, editing, and deleting subjects in the system. | Lists all subjects with their details including subject code, subject name, and description. Displays active/inactive status. Provides options to create, edit, view details, or delete subjects. Includes filtering by code and status, sorting, and export functionality (PDF, Excel, CSV, Image). Validates subject code uniqueness. All operations include validation and confirmation dialogs where appropriate. |
| **6.0 Curriculum Management** | User interface for viewing, creating, editing, and deleting curriculum programs in the system. | Displays all curriculums with their details including curriculum name, code, academic year, semester, grade level, and description. Shows statistics for each curriculum including total subjects, teachers, and enrolled students. Provides options to create, edit, view details, or delete curriculums. Includes filtering by grade level, academic year, and status. Supports assigning subjects with teachers and enrolling students to curriculums. All operations include validation and confirmation dialogs where appropriate. |

## Teacher Branch Screens

| Screen | Description | REMARKS |
|--------|-------------|---------|
| **7.0 Teacher Dashboard** | Main dashboard for teachers displaying their assigned subjects, classes, and overview statistics. | Displays overview of teacher's assigned subjects, upcoming classes, assessment statistics, and recent activities. Shows statistics including total assigned subjects, total students across all subjects, pending assessments, and overall performance metrics. Provides navigation to Subject Information Page and Assessment Records Page. |
| **8.0 Subject Information Page** | User interface for teachers to view detailed information about their assigned subjects. | Lists all subjects assigned to the logged-in teacher. Displays subject details including subject code, subject name, description, curriculum information, enrolled students count, and class schedule. Allows teachers to view student lists for each subject. Provides navigation to Assessment Records for each subject. |
| **9.0 Assessment Records Page** | User interface for teachers to view, create, and manage assessment records for their subjects. | Displays assessment records organized by subject. Shows assessment details including assessment type, date, subject, students assessed, and grades. Allows teachers to filter by subject, date range, and assessment type. Provides "Add New Assessment" button that opens a modal (9.1) to create new assessments. Includes export functionality for assessment reports. |
| **9.1 Add New Assessment Modal** | Modal dialog for teachers to create new assessment records. | Opens from Assessment Records Page (9.0) when "Add New Assessment" button is clicked. Allows teachers to input assessment details including assessment type, subject selection, date, description, and student grades. Supports bulk grade entry for multiple students. Includes validation and save functionality. Closes modal and refreshes Assessment Records Page upon successful creation. |

## Student Branch Screens

| Screen | Description | REMARKS |
|--------|-------------|---------|
| **10.0 Student Dashboard** | Main dashboard for students displaying welcome message, academic summary, and enrolled subjects. | Displays personalized welcome banner with student name and grade level. Shows academic summary cards including Overall Average, Subjects count, and Academic Year. Lists all enrolled subjects in "My Subjects" section with subject name, code, semester, academic year, description, and final grade. Provides navigation to My Grades, Student Profile, and Student Report screens. |
| **11.0 My Grades** | User interface for students to view their academic grades by course with grade components. | Displays "My Grades" banner with student name and grade level. Shows "Academic Grades" section with term filter dropdown (All Terms, Prelim, Midterm, Semifinal, Final). Lists courses with course name, course code, and expandable grade components (Prelim, Midterm, Semifinal, Final, Final Grade). Each course card can be expanded/collapsed to view grade breakdown. Shows individual grades for each period or dash if not yet graded. |
| **12.0 Student Profile** | User interface for students to view and manage their account information and profile. | Displays "Student Profile" banner with subtitle. Shows "Profile Information" section with profile picture, "Change Photo" button, and read-only fields: Full Name, Username, Student ID, Curriculum, and Academic Year. Provides "Edit" button to enable editing of profile information. Profile picture can be changed by uploading new image. |
| **13.0 Student Report** | User interface for students to generate and view their academic report. | Displays "Student Report" banner with student name. Shows term selection dropdown (All Periods, Prelim, Midterm, Semifinal, Final) and "Generate Report" button. Displays report table with columns: Course, Prelim, Midterm, Semifinal, Final, and Teacher. Lists all enrolled courses with course codes, grade periods, and assigned teacher names. Shows grades for each period or dash if not yet graded. Supports filtering by term and generating reports. |
